System.Net.Sockets Library

The System.Net.Sockets library contains classes providing client and server side synchronous network socket connections. The fundamental class is the TCPSocket class, that represents a synchronous reliable connection oriented socket.

Client side connection can be established for example by constructing a TCPSocket with a host name or IP address, and a port number or a service name by using this constructor and then send and receive data.

An alternative way is to construct a TCPClient and then call the GetStream member function to get a network stream. The network stream can be read and written directly or it can be passed further to a StreamReader and/or a StreamWriter for text I/O, or to a BinaryReader and/or a BinaryWriter for binary I/O. A client side socket has upport for HTTPS connections implemented by means of the GnuTLS library. HTTPS support should be considered experimental.

Server side connection is established by first binding a TCPSocket to a port, then putting the socket to listening mode and finally accepting client connections.

There are limitations with the server side sockets: they do not support HTTPS connections (yet) and they do not scale to thousands of connections or more because lack of asynchrony. The programming model is thread per client.

A TCPListener is a simple wrapper class for a server side TCPSocket.
