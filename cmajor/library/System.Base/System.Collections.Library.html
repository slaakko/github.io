System.Collections Library

The System.Collections library contains container classes.

List

The most fundamental container is the List class that is like an array that can grow. Elements can be added to it either by copying or moving. When moving an element you can use the Rvalue function that converts its argument to an rvalue. Adding an element to the end of the list takes amortized constant time. Elements of a list can be accessed by index using the subscript operator. Element access is O(1) operation.

Set

Set is a collection of unique elements that are kept sorted in a red-black tree. By default the sorting operation is the less-than operation, but you can give other ordering relation as the second template argument. Searching for an element in the set and inserting an element to the set are Olog(n) operations.

Map

Map is an associative array of key-value pairs that are kept sorted by keys in a red-black tree. By default the keys are sorted by the less-than operation, but you can give your own sorting criteria as the third template argument. Searching for an element in the map and inserting an element to the map are Olog(n) operations.

HashSet

HashSet is a collection of unique elements that are organized in a hashtable. The elements need not be ordered, but you must be able to compare them for equality or other equality-like relation provided as the third template argument. The second template argument is a hash function object that is by default the Hasher function object that delegates the computation of the hash function for the element to the GetHashCode function. If there is not a GetHashCode overload for the your element type, you can provide your own, and it should work with the standard Hasher. Adding an element to the hash set depends on the hash function, but can be thought as O(1) operation. Searching for an element in a hash set depends on the hash function, but can be thought as O(1) operation.

HashMap

HashMap is an associative array of key-value pairs that are organized in a hashtable. The keys need not be ordered, but you must be able to compare for equality or other equality-like relation provided as the fourth template argument. As for HashSet, the third template argument is a hash function for the key type, by default Hasher function object, that delegates the computation of the hash function to the GetHashCode function. Adding an element to the hash map depends on the hash function, but can be thought as O(1) operation. Searching for an element in a hash map depends on the hash function, but can be thought as O(1) operation.

LinkedList

LinkedList is a doubly linked list of elements. Inserting an element to the linked list in a known position is O(1) operation. Searching for an element in a linked list is O(n) operation.

ForwardList

ForwardList is a singly linked list of elements. Inserting an element to the forward list in a known position is O(1) operation. Searching for an element in a forward list is O(n) operation.

Stack

Stack is a last-in-first-out container of elements implemented as a List. Pushing an element to the top of the stack takes amortized constant time. Popping an element from the top of the stack is O(1) operation.

Queue

Queue is a first-in-first-out container of elements implemented as a List. Putting an element to the end of the queue takes amortized constant time. Getting an element from the start of the queue is O(n) operation.

Bitset

Bitset is a variable length container of bits implemented as a List of ulongs. Setting, resetting and accessing a bit takes constant time.
