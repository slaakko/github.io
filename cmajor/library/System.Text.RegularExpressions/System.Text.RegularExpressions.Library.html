System.Text.RegularExpressions Library

The System.Text.RegularExpressions library contains an implementation of a simple regular expression matching engine.

RegEx

The RegEx class contains the following basic functionality:

  • RegEx constructor constructs a regular expression from the given pattern that is not compiled to a DFA, and the pattern must match entirely the input.
  • RegEx constructor takes also flags that can be ORed together. They allow the regular expression (1) to be compiled to a DFA and (2) match within a line of input instead of the entire input.
  • The Matches member function matches an input string to the pattern contained by the regular expression and returns true if input matches, and false otherwise.

The regular expression is represented inside the engine as an NFA or it can be compiled to a DFA. Compiling to DFA takes somewhat longer than construction of an NFA but matching is faster with DFA than with NFA.

Syntax

The following table contains the recognized regular expression pattern syntax:

regular‑expression → alternative‑expr
alternative‑expr → sequence‑expr (| sequence‑expr)*
sequence‑expr → postfix‑expr postfix‑expr*
postfix‑expr → primary‑expr (* | + | ?)?
primary‑expr → char | . | char‑class | ( alternative‑expr ) | \d | \D | \s | \S | \w | \W | \l | \L | \u | \U | escape
char → any‑char − [()\\\[*+?.]
any‑char → 'any character'
char‑class → [ (class | range)* ]
class → [:alnum:] | [:alpha:] | [:blank:] | [:cntrl:] | [:digit:] | [:graph:] | [:lower:] | [:print:] | [:punct:] | [:space:] | [:upper:] | [:word:] | [:w:] | [:xdigit:]
range → class‑char (− class‑char)?
class‑char → [\\\]] | escape
escape → \ [xX]hex | octal | [dD]decimal | [abfnrtv0]
hex → hex‑digit+
hex‑digit → [0-9a-fA-F]
octal → octal‑digit octal‑digit octal‑digit
octal‑digit → [0-7]
decimal → decimal‑digit+
decimal‑digit → [0-9]
